# T4 — Client Portal Prompt

## Dono da tranche
Zeus

## Executor principal
Vulcanus

## Missão
Entregar o portal privado do cliente com o mínimo funcional necessário para onboarding e relação contínua.

## Entregáveis mínimos
- login do cliente
- dashboard inicial
- checklist de onboarding
- upload de documentos
- timeline ou histórico simples
- recommendation ledger inicial
- flags internas de pendência

## Sucesso
A tranche fecha quando:
1. um cliente beta consegue acessar
2. consegue enviar documentos
3. Bruno consegue acompanhar pendências
4. uma recomendação ou memo fica registrada no ledger

## Fracasso
A tranche falha se:
- o portal for só uma casca visual
- o cliente não conseguir completar ações reais
- documentos não tiverem trilha mínima
- o ledger não existir de verdade

## Instrução para Vulcanus
Faça um portal austero, confiável e útil.
