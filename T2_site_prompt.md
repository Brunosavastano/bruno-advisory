# T2 — Site & Intake Prompt

## Dono da tranche
Zeus

## Executor principal
Vulcanus

## Missão
Colocar no ar a superfície pública do negócio e fazer o intake funcionar de ponta a ponta.

## Entregáveis mínimos
- homepage
- páginas institucionais essenciais
- CTA principal
- formulário de fit
- captura do lead na DB
- analytics/eventos básicos
- visualização interna dos leads recebidos

## Sucesso
A tranche fecha quando:
1. um visitante consegue entender a proposta
2. consegue enviar seus dados
3. esses dados entram na DB do projeto
4. Bruno consegue ver o lead no painel interno
5. há evidência de fluxo ponta a ponta testado

## Fracasso
A tranche falha se:
- o site estiver bonito, mas sem intake real
- o formulário não persistir
- o lead não aparecer internamente
- analytics estiverem cegos

## Instrução para Vulcanus
Otimize para fluxo funcional, não para design ornamental.
